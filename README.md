# 🎯 Resonance - Платформа автоматизации подбора молодых специалистов

## 📋 Описание проекта

Resonance — это backend-сервис для автоматизации подбора молодёжных кадров, решающий ключевую проблему современного рынка труда: дефицит молодых специалистов при растущем спросе на них.

### Проблематика

Рынок труда сталкивается с рядом критических вызовов:

- **Растущий дефицит молодых специалистов** при ежегодно увеличивающемся спросе
- **Увеличение сроков закрытия вакансий**: ранее 88% позиций заполнялись менее чем за два месяца, сегодня требуется более трёх месяцев
- **Низкие конверсии** на этапах подбора (1–5%), указывающие на неэффективность текущих процессов найма
- **Отсутствие системной аналитики** и единых инструментов оценки кандидатов
- **Отсутствие прозрачной кадровой воронки** и высокие затраты на закрытие вакансий

### Решение

Resonance предоставляет комплексное решение для структурирования процесса найма молодых специалистов:

✅ **Структурирование процесса найма** - единая система управления кандидатами  
✅ **Повышение конверсий** - оптимизация каждого этапа воронки подбора  
✅ **Сокращение сроков** закрытия вакансий  
✅ **Снижение затрат** на рекрутинг  
✅ **Системная аналитика** - прозрачность и метрики подбора  
✅ **Единая база данных** молодёжных кадров для оперативного поиска кандидатов

### Целевая аудитория

**Молодые специалисты** получают возможность быстро найти работу благодаря:
- Готовности быстро учиться
- Высокой вовлечённости
- Ориентированности на развитие
- Способности адаптироваться к новым задачам

**Компании** получают доступ к структурированной базе перспективных кандидатов и инструменты для эффективного найма.

## 🏗️ Архитектура и технологии

Проект построен на базе фреймворка **Spring Boot** с использованием монолитной слоистой архитектуры.

### 📚 Стек технологий

**Backend:**
- **Kotlin** - современный язык программирования для JVM
- **Spring Boot 3.3.5** - фреймворк для создания Java-приложений
- **Spring Security** - защита приложения и аутентификация
- **Spring Data JPA** - работа с базой данных через ORM
- **JWT (JSON Web Tokens)** - stateless аутентификация пользователей

**База данных:**
- **PostgreSQL 15** - реляционная СУБД для хранения данных

**Инфраструктура:**
- **Docker & Docker Compose** - контейнеризация и оркестрация сервисов
- **Gradle** - система сборки проекта

**Дополнительные инструменты:**
- **Spring Boot Actuator** - мониторинг и health checks
- **Jackson** - сериализация/десериализация JSON
- **Hibernate** - ORM для работы с БД

## 🚀 Быстрый старт

### Предварительные требования

- **Docker** 20.10+
- **Docker Compose** 2.0+
- **Git**

### Установка и запуск

#### 1. Клонируйте репозиторий

```bash
git clone https://github.com/your-username/resonance.git
cd resonance
```

#### 2. Создайте файл окружения

Отредактируйте `.env` и установите **уникальные JWT ключи**:

```bash
# Генерация безопасных ключей (Linux/MacOS):
openssl rand -base64 64

# Windows PowerShell:
[Convert]::ToBase64String((1..64 | ForEach-Object { Get-Random -Maximum 256 }))
```

Вставьте сгенерированные ключи в `.env`:

```env
JWT_SECRET=your-generated-secret-key-here
JWT_REFRESH_SECRET=your-generated-refresh-secret-key-here
```

#### 3. Запустите проект

```bash
# Сборка и запуск всех сервисов
docker compose up -d

# Просмотр логов
docker compose logs -f app

# Проверка статуса
docker compose ps
```

#### 4. Проверьте работу

```bash
# Health check
curl http://localhost:8088/actuator/health

# Ожидаемый ответ:
# {"status":"UP","groups":["liveness","readiness"]}
```

## 📁 Структура проекта

Проект организован в несколько слоёв, каждый из которых отвечает за свою область ответственности.

### Описание слоёв

| Слой | Ответственность |
|------|-----------------|
| **Controller** | Приём HTTP-запросов, валидация входных данных |
| **Service** | Бизнес-логика приложения |
| **Database** | Взаимодействие с базой данных через JPA, управление репозиториями |
| **Model** | Модели данных (Entity и DTO) |
| **Security** | JWT аутентификация и авторизация |
| **Logger** | Логирование событий, трейсинг, аудит операций |
| **Errors** | Централизованная обработка ошибок и исключений |

## 🔐 Система безопасности

### JWT Authentication

Приложение использует **JWT (JSON Web Tokens)** для stateless аутентификации:

- **Access Token** - короткоживущий токен (15 минут) для доступа к API
- **Refresh Token** - долгоживущий токен (7 дней) для обновления access token

### Роли пользователей

| Роль | Описание | Доступ |
|------|----------|--------|
| **STUDENT** | Молодой специалист | Просмотр и управление своим профилем |
| **COMPANY** | Работодатель | Управление вакансиями, просмотр кандидатов |

## 🗃️ База данных

### Подключение к БД

```bash
# Из Docker
docker compose exec postgres psql -U res -d resonance

# С хост-машины
psql -h localhost -p 5432 -U res -d resonance
```

**Параметры:**
- Host: `localhost` (или `postgres` внутри Docker)
- Port: `5432`
- Database: `resonance`
- Username: `res`
- Password: `pass`
